## Phase 3: Integration & Polish

### 3.1 Cross-Linking

**Make sure all pieces reference each other**

**Pattern**:

- Notebook references formal paper (for rigor)
- Formal paper references notebook (for code)
- Curriculum references both (for teaching)
- README ties everything together

**Checklist**:

- [ ] Every document links to related documents
- [ ] Navigation is clear (readers don't get lost)
- [ ] Progressive depth paths exist (beginner ‚Üí expert)
- [ ] No orphaned content

### 3.2 Visual Coherence

**Make it LOOK like it belongs together**

**Elements**:

- Consistent headers (H&&S:WAVE, ATOM tags, session IDs)
- Unified color scheme (if applicable)
- Recurring symbols/metaphors
- Signature sign-off ("From the constraints, gifts...")

**This creates RECOGNITION**: "Oh, this is SpiralSafe work"

### 3.3 Accessibility Check

**Can each audience find their entry point?**

**Test**:

- Hand formal paper to a mathematician - do they see rigor?
- Hand notebook to a coder - can they run it?
- Hand curriculum to a teacher - could they teach tomorrow?
- Hand simple story to a child - do they understand?

**If any fail, iterate**.

### 3.4 The "So What?" Test

**Does this answer the question it set out to answer?**

**Checklist**:

- [ ] Core claim is proven/demonstrated
- [ ] Evidence is compelling
- [ ] Implications are clear
- [ ] Next steps are actionable
- [ ] Reader feels "aha!"

**Rationale**: Serve both audiences. Coders run notebook, non-coders read paper.

**Result**: Worked perfectly. Each audience got what they needed.

## Prompts for Different Scenarios

### Scenario 1: Theoretical Framework

**You want to**: Develop a new mathematical or conceptual framework

**Questions to ask**:

1. What's the minimal set of axioms/definitions?
2. What are the core theorems/claims?
3. What's a concrete example that proves it works?
4. How does this connect to existing theory?
5. What are the open questions?

**Deliverables**:

- Formal paper with proofs
- Jupyter notebook with examples
- Visualization of key concepts

### Scenario 2: Practical Tool/System

**You want to**: Build a tool/system that solves a real problem

**Questions to ask**:

1. What problem does this solve? (Be specific)
2. Who are the users?
3. What's the minimal viable version?
4. How do users learn it?
5. How do we measure success?

**Deliverables**:

- Working prototype (code/build)
- User guide / tutorial
- API documentation
- Example use cases

### Scenario 3: Educational Content

**You want to**: Teach a complex topic accessibly

**Questions to ask**:

1. What's the core insight in one sentence?
2. What's the simplest true thing I can say?
3. What's a concrete activity students can do?
4. How do I scaffold from simple ‚Üí complex?
5. How do I assess understanding?

**Deliverables**:

- Curriculum with lesson plans
- Student worksheets
- Teacher guide
- Assessment rubrics

### Scenario 4: Research Investigation

**You want to**: Explore an open question deeply

**Questions to ask**:

1. What exactly am I trying to prove/discover?

## Common Pitfalls

### Pitfall 1: Premature Organization

**Symptom**: Spending hours on outline before understanding structure

**Fix**: Free-write first. Structure emerges naturally.

### Pitfall 2: Single-Audience Trap

**Symptom**: Writing only for experts or only for beginners

**Fix**: Create layered content. Serve all audiences.

### Pitfall 3: Perfectionism Paralysis

**Symptom**: Never releasing because "not ready yet"

**Fix**: Ship at 80% complete. Iterate based on feedback.

### Pitfall 4: Lost in Abstraction

**Symptom**: Beautiful theory with no concrete examples

**Fix**: Find anchor example first. Theory follows.

### Pitfall 5: Scope Creep

**Symptom**: Deep-dive expands to cover everything

**Fix**: Define scope in Phase 0. Stay focused. Note expansions as "Future Work."
